Energy Calculation Methodology

This methodology estimates the energy consumption and CO2 emissions of large language model (LLM) inference using a FLOP-based energy model derived from Epoch AI's research on ChatGPT energy consumption. The approach models both the prefill phase (processing input tokens) and decode phase (generating output tokens) with different computational characteristics, updated to reflect estimated GPT-5 specifications and modern datacenter infrastructure.

1. Model Parameters (GPT-5 Proxy)

Updated Model Specifications:

  • Total Parameters: 300 billion (used for memory residency estimation)
  • Active Parameters: 101.17 billion (33.7% activation ratio for MoE architecture)
  • Attention Coefficient (a): 2,635,100 (derived from 4√óL√óh_q√ód_h scaling)
  • Architecture: Mixture of Experts (MoE) with sparse activation

Note: These parameters represent estimates for a GPT-5 class model, scaling from known GPT-4o characteristics (estimated 200B total parameters, ~100B active) to reflect next-generation capabilities.

2. Hardware Infrastructure (Modern Datacenter Setup)

GPU Specifications (H100-class):

  • Thermal Design Power: 700W per GPU
  • Total Power per GPU: ~1,500W (including server overhead, cooling, networking)
  • Compute Capacity: 1.979√ó10¬π‚Åµ FLOP/s per GPU (BF16 precision, theoretical peak)
  • Practical Utilization: 10% for inference workloads (memory-bound operations)
  • Memory: 80 GB per GPU
  • Weight Quantization: 4-bit precision for memory calculations

Datacenter Infrastructure:

  • Power Usage Effectiveness (PUE): 1.2
  • Server Configuration: 8 GPUs per server node
  • Server Overhead: 800W per 8-GPU node (excluding GPU power)
  • Average Power Utilization: 70% of peak power consumption

Based on NVIDIA DGX H100 server specifications and industry-standard datacenter configurations.

3. FLOP Estimation Model

Following the Epoch AI methodology, computational workload is divided into two phases:

Prefill Phase (Input Processing):

Where:

  • N_active = Active parameters (101.17B for GPT-5 proxy)
  • T_in = Input token count

  • Accounts for quadratic scaling of attention mechanisms with input length
  • Separates prefill and decode computational costs

  • Typical Input: Variable, from short prompts (~50 tokens) to long documents (10k-100k tokens)

7. Scaling Characteristics

Input Length Scaling:

  • Short queries (50 tokens in, 500 tokens out): ~0.3 Wh
  • Medium documents (10k tokens in, 500 tokens out): ~2.5 Wh
  • Large documents (100k tokens in, 500 tokens out): ~40 Wh

Note: Input processing cost is incurred once per conversation due to KV caching.

8. Key Assumptions and Limitations

Research-Based Assumptions:

  • Hardware Utilization: 10% effective utilization for inference (Epoch AI finding)
  • Power Scaling: 70% average power consumption vs. peak rating
  • Model Architecture: MoE with 33.7% parameter activation ratio
  • Datacenter Efficiency: Modern H100-based infrastructure

Known Limitations:

  • Token Estimation: Character-based approximation vs. actual tokenization
  • Model Scaling: GPT-5 parameters are estimated, not confirmed
  • Infrastructure Variation: Actual datacenter configurations may differ
  • Optimization Techniques: Advanced serving optimizations not modeled
